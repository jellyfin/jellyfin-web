# Jellyfin Security Check Workflow

## الخطوات:
1. تشغيل فحص الثغرات الأمنية: `npm audit`
2. فحص التبعيات المحدثة: `npm outdated`
3. تحليل النتائج وتحديد الأولويات
4. تحديث الحزم الآمنة: `npm update`
5. إنشاء تقرير بالثغرات المتبقية
6. اقتراح خطة عمل للثغرات الحرجة

## ملاحظات:
- ركز على الثغرات الحرجة والعالية أولاً
- تأكد من اختبار التطبيق بعد التحديثات
- وثق أي تغييرات مهمة في CHANGELOG
